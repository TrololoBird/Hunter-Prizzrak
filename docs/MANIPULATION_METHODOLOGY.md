# SMC Dump-формации — Методология по транскрипции (IMG_2700)

## Последовательность (только из транскрипции)

```
1. Анализ со старших таймфреймов, переход к младшим

2. Монета в восходящем канале, постоянные импульсы,
   каждый импульс поглощается

3. Ждать финальный импульс — обновление ДВУХ максимумов (снятие ликвидности)

4. При обновлении максимума — сразу набирать частично позиции в шорт,
   если выше ликвидности нет

5. После снятия хая — перейти на младшие таймфреймы искать точку входа

6. Мощный памп (снятие ликвидности) → свечи затухают

7. После затухания: свеча с импульсом вниз, закрылась красной

8. После красной: сформировали боковик локальный

9. Уже в позиции от самого пика

10. Подтверждение на младшем таймфрейме → движение вниз
```

## Что известно о входе

- Вход: частично при обновлении максимума, «от самого пика»
- Условие: нет ликвидности выше
- Свип = обновление двух максимумов = «мощный памп»

## Что известно о целях

- «30% чистого движения цены» (результат)
- «Следующая цель — 0.2034» (для GTC)

## Чего в транскрипции нет

- ATR, объём, BOS/CHoCH, стоп
- fractals, scoring, WS/REST
- формулы, числа, tolerance, проценты распределения

## research_manipulations.py — отличия от транскрипции

| В транскрипции | В коде |
|----------------|--------|
| Обновление двух максимумов (EQH) | Свип ближайшего по времени хая |
| Нет ликвидности выше | Не проверяется |
| Затухание свечей | 3+ свечи с телом ≤70% |
| Импульс вниз, красная | displacement: красная > avg10, close < prev_low |
| Боковик локальный (после красной) | Не детектится |
| Подтверждение на младшем ТФ | Не реализовано |
| Частичный вход на пике | Одностадийный вход на пике |
